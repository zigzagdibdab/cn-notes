Network Types
Unit 1 · LAN · MAN · WAN

Networks are classified by their geographic coverage. Three fundamental categories form the backbone of this classification:

LAN

Local Area Network. Covers a room, building, or campus. Speeds: 100 Mbps – 10 Gbps.

≤ 1 km
MAN

Metropolitan Area Network. Spans a city or metro area. Used by ISPs and cable operators.

1 – 100 km
WAN

Wide Area Network. Spans countries/continents. Uses leased lines, satellites, undersea cables.

> 100 km

Key insight The Internet is not a WAN itself — it is an internetwork (network of networks), composed of thousands of LANs, MANs, and WANs interconnected by routers.
Wired vs Wireless & Broadcast
Unit 1 · Transmission modes

Wired Networks

Use physical media (copper wire, fiber optic, coaxial cable) to carry signals. Offer higher security, reliability, and speed but limited mobility.

Wireless Networks

Use electromagnetic waves (radio, microwave, infrared). Provide mobility and easy deployment but susceptible to interference, eavesdropping, and shared-medium collisions.

Broadcast Network

Single communication channel shared by all. Every device receives every packet; address field determines the intended recipient.

e.g., Wi-Fi, Cable TV
Point-to-Point Network

Dedicated link between exactly two endpoints. More secure, no collision. Multiple hops may be needed to reach destination.

e.g., WAN leased line
Unicast

One sender → one specific receiver.

Multicast

One sender → a group of receivers.

Broadcast

One sender → all devices on the network.

Unit 1 · Physical & Logical arrangement

Topology describes the geometric arrangement of nodes and links. There is a difference between physical topology (actual cable layout) and logical topology (how data flows).

Data & Signals
Unit 2 · Physical Layer
Analog Signal

Continuous waveform. Described by amplitude, frequency, and phase. Example: voice, AM/FM radio.

Digital Signal

Discrete voltage levels (0s and 1s). Easier to regenerate. Used in computers and modern networks.

Periodic Signal

Repeats in a fixed time period T. Frequency f = 1/T (Hz).

Aperiodic Signal

Does not repeat. Most data signals are aperiodic.

Key Parameters

Parameter Definition Unit
Amplitude Peak value/strength of signal Volts
Frequency Number of cycles per second Hz
Period Time for one complete cycle (T = 1/f) Seconds
Phase Position of waveform relative to reference Degrees
Wavelength λ = v/f (distance one cycle travels) Meters
Bandwidth Range of frequencies a channel can carry Hz

Nyquist & Shannon Theorems

// Nyquist Theorem (Noiseless channel) Max Data Rate = 2 × Bandwidth × log₂(L) L = number of signal levels // Shannon's Capacity Theorem (Noisy channel) Capacity = Bandwidth × log₂(1 + SNR) SNR = Signal-to-Noise Ratio (in dB: SNRdb = 10·log₁₀(SNR))
Exam tip Shannon gives the theoretical maximum; Nyquist gives the maximum for a specific encoding scheme. Real systems are always limited by the Shannon bound.

Digital-to-Digital Encoding

Scheme Description Notes
NRZ-L Level: 0 = high, 1 = low (or vice versa) Simple but no sync for long runs

Analog-to-Analog Modulation (Carriers)

AM — Amplitude Modulation

Signal info encoded in amplitude of carrier wave. Susceptible to noise.

FM — Frequency Modulation

Info encoded in frequency shifts. Less noise. Used in FM radio.

PM — Phase Modulation

Info encoded in phase shifts of carrier.

Digital-to-Analog Modulation

ASK

Amplitude Shift Keying — carrier amplitude changes per bit. Low noise immunity.

FSK

Frequency Shift Keying — frequency changes. Better noise immunity. Used in modems.

PSK

Phase Shift Keying — phase changes. BPSK (2 phases), QPSK (4 phases).

QAM

Quadrature AM — combines ASK + PSK. Very efficient. Used in DSL, 5G.

Analog-to-Digital (PCM)

Pulse Code Modulation has 3 steps: Sampling (at ≥ 2× max frequency per Nyquist), Quantization (assign discrete levels), Encoding (convert to binary).

Multiplexing
Unit 2 · FDM · TDM · WDM

Multiplexing combines multiple signals onto one shared medium to maximize bandwidth efficiency. A multiplexer (MUX) combines signals at the sender; a demultiplexer (DEMUX) separates them at the receiver.

FDM — Frequency Division

Each signal uses a different frequency band (channel). Channels are separated by guard bands. Used in AM/FM radio, cable TV, ADSL.

Analog
TDM — Time Division

All signals share the full bandwidth but in different time slots. Two types: Synchronous (fixed slots) and Statistical/Asynchronous (dynamic).

Digital
WDM — Wavelength Division

Multiple optical signals at different wavelengths on a single fiber. DWDM (Dense WDM) packs many channels. Used in fiber optic backbones.

Optical
CDM — Code Division

Each sender uses a unique code (chip sequence). All transmit simultaneously on same frequency. Used in CDMA mobile networks.

Switching Techniques
Unit 2 · Circuit · Message · Packet

Switching determines how data is routed through a network. Three fundamental approaches exist:

Circuit Switching

A dedicated physical path is established before communication. Resources are reserved for the entire duration. Example: Traditional PSTN telephone.

3 Phases: Setup → Transfer → Teardown
Message Switching

Entire message is stored at each intermediate node, then forwarded. Store-and-forward. No dedicated path. Old telegraph networks.

Store and Forward
Packet Switching

Message is broken into packets. Each packet independently routed. Two types: Datagram (connectionless, IP) and Virtual Circuit (connection-oriented, ATM).

The Internet uses this
Feature Circuit Switching Packet Switching
Dedicated path Yes No
Bandwidth Fixed, reserved Dynamic, shared
Delay Low (after setup) Variable (queuing)
Efficiency Low (idle waste) High
Failure handling Re-setup required Rerouting per packet
Example PSTN phone call Internet (IP), Wi-Fi
Data-Link Control (DLC)
Unit 3 · Framing · Error Control · Flow Control

The Data-Link Layer is split into two sublayers in IEEE architecture: LLC (Logical Link Control) and MAC (Media Access Control).

DLC Services

  • Framing — Encapsulates packets into frames. Delimited by flags or character counts.
  • Error Control — Detects and/or corrects errors (CRC, Hamming code, parity).
  • Flow Control — Prevents fast sender from overwhelming slow receiver.

Error Detection Methods

Method How it works Detects
Parity Bit One extra bit to make total 1s even/odd Single bit errors only
CRC (Cyclic Redundancy Check) Polynomial division; remainder appended to frame Burst errors; very powerful
Checksum Sum of all bytes; complement appended Used in TCP/UDP/IP headers
Hamming Code Uses redundant parity bits at power-of-2 positions Detects AND corrects single-bit errors

Flow Control Methods

  • Stop-and-Wait — Send one frame, wait for ACK. Simple but inefficient for high-delay links.
  • Sliding Window — Multiple frames in flight simultaneously. Window size = how many unACKed frames allowed.
Efficiency formula For Stop-and-Wait: Efficiency = 1 / (1 + 2a) where a = propagation delay / transmission time. Large 'a' (satellite links) → very inefficient → need larger window sizes.
Multiple Access Protocols
Unit 3 · MAC Sublayer

When multiple stations share a broadcast medium, a Multiple Access Protocol coordinates who transmits when to avoid collisions.

Three Categories

1. Random Access

Protocol Description
ALOHA Transmit whenever ready. If collision, wait random time and retry. 18% efficiency max.
Slotted ALOHA Sync to slots. Up to 37% efficiency.
CSMA Carrier Sense: listen before transmit. Reduces but doesn't eliminate collisions.
CSMA/CD +Collision Detection: abort on collision, jam signal, backoff. Used in Ethernet.
CSMA/CA +Collision Avoidance: IFS + backoff before transmit. Used in Wi-Fi (802.11).

2. Controlled Access

  • Reservation — Stations reserve time slots in a mini-frame.
  • Polling — Central controller (primary) polls secondary stations in turn.
  • Token Passing — A token circulates; only the token holder can transmit. Used in Token Ring, FDDI.

3. Channelization

  • FDMA — Each station gets a frequency band. Used in first-gen mobile.
  • TDMA — Each station gets time slots. Used in GSM.
  • CDMA — Each station uses a unique code. Used in 3G/WCDMA.
Wired LANs — Ethernet

Ethernet (IEEE 802.3) is the dominant wired LAN technology. Originally used a bus topology with CSMA/CD; modern Ethernet uses full-duplex switched star topology — no collisions.

MAC Address

A 48-bit (6-byte) hardware address burned into NIC. Format: AA:BB:CC:DD:EE:FF. First 3 bytes = OUI (manufacturer ID). Last 3 bytes = device ID. Broadcast address: FF:FF:FF:FF:FF:FF

Wi-Fi MAC — CSMA/CA + DIFS/SIFS

  • Cannot use CSMA/CD (can't detect collisions while transmitting wirelessly)
  • Uses CSMA/CA with IFS (Interframe Space): SIFS (short), DIFS (distributed)
  • Hidden Station Problem — solved by RTS/CTS mechanism
  • Exposed Station Problem — a station unnecessarily defers
  • Modes: Infrastructure (with AP) and Ad hoc / IBSS (direct)

Bluetooth (IEEE 802.15.1)

  • Short range (<10 m), 2.4 GHz, uses FHSS (Frequency Hopping Spread Spectrum)
  • Forms a piconet: 1 master + up to 7 active slaves
  • Multiple piconets form a scatternet
  • Versions: Bluetooth 1.x (1 Mbps), 2.x EDR (3 Mbps), 4.x BLE (low energy), 5.x (2 Mbps+)
  • Layers: Radio → Baseband → L2CAP → Application protocols
WiMAX · Cellular · Satellite · Mobile IP
Unit 3 · Wide-area Wireless

WiMAX (IEEE 802.16)

Wireless MAN technology. Up to 70 Mbps, range ~50 km. Uses OFDM. Designed as a "last mile" broadband alternative. Fixed WiMAX (802.16d) and Mobile WiMAX (802.16e).

Cellular Networks

Generation Key Tech Data Speed

Satellite Networks

  • GEO (36,000 km): Weather, TV broadcast, GPS-like systems. ~270ms propagation delay.
  • MEO (5,000-15,000 km): GPS satellites (20,200 km). Medium delay.
  • LEO (200-1,400 km): Low latency (~5-15ms), Starlink, Iridium. Need many satellites for coverage.

Mobile IP

Allows a mobile device to keep its IP address as it moves across networks. Key entities: Home Agent (HA), Foreign Agent (FA), Care-of Address (CoA). Process: Device registers CoA with HA → HA tunnels packets to FA → FA delivers to device.

Transport Layer — Error Control Protocols
Unit 4 · ARQ Protocols

Automatic Repeat Request (ARQ) protocols handle errors by requesting retransmission. Four protocols from simplest to most efficient:

1. Simple (Unreliable)

No error control, no flow control. Sender just transmits; receiver just receives. No ACKs. Suitable only if no errors expected (or handled at higher layer). Used conceptually to understand the baseline.

2. Stop-and-Wait ARQ

  • Send one frame → wait for ACK before sending next
  • If ACK not received in timeout period → retransmit
  • Sequence numbers: 0 and 1 only (1 bit)
  • Problem: Very poor efficiency when propagation delay is large
Efficiency η = 1 / (1 + 2a) where a = Tp / Tf (propagation time / frame transmission time)

3. Go-Back-N (GBN)

  • Sender can have up to 2ⁿ - 1 unACKed frames in flight (window size)
  • On error/timeout: retransmit the errored frame AND ALL subsequent frames
  • Receiver only accepts in-order frames; discards out-of-order
  • Receiver window size = 1
  • Simple receiver logic but wastes bandwidth on retransmissions
GBN Window Constraint Window size ≤ 2ⁿ - 1 (for n-bit sequence numbers). If = 2ⁿ, receiver can't distinguish new frame from retransmitted old frame.
Selective Repeat & Bidirectional
Unit 4 · SR · HDLC · PPP

Selective Repeat (SR)

  • Only the specific errored frame is retransmitted (not all subsequent)
  • Receiver has a buffer to hold out-of-order frames
  • Both sender AND receiver window size = 2ⁿ/2 = 2ⁿ⁻¹
  • More efficient than GBN but more complex receiver
Feature Stop-and-Wait Go-Back-N Selective Repeat
Sender Window 1 2ⁿ-1 2ⁿ⁻¹
Receiver Window 1 1 2ⁿ⁻¹
Retransmission 1 frame All from error Only errored
Buffer needed No No (receiver) Yes (receiver)
Efficiency Low Medium High

Bidirectional Protocols (HDLC)

HDLC (High-level Data Link Control) supports full-duplex. Three modes:

  • NRM (Normal Response Mode) — Primary/secondary; secondary only responds
  • ABM (Asynchronous Balanced Mode) — Both peers equal; used in PPP
  • ARM (Asynchronous Response Mode) — Secondary can initiate

PPP (Point-to-Point Protocol) — Common WAN link protocol; carries IP over serial/DSL/phone links. Uses LCP for setup and NCP for network layer config.

UDP & TCP
Unit 4 · Internet Transport Protocols

UDP — User Datagram Protocol

Connectionless

No handshake. Send and forget. Faster but unreliable.

No Ordering

Datagrams may arrive out of order. No retransmission.

Uses

DNS, DHCP, VoIP, video streaming, online gaming — where speed > reliability.

// UDP Header (8 bytes only) | Source Port (16) | Dest Port (16) | | Length (16) | Checksum (16) |

TCP — Transmission Control Protocol

  • Connection-oriented: 3-way handshake (SYN → SYN-ACK → ACK)
  • Reliable: Sequence numbers, acknowledgments, retransmission on timeout
  • Ordered delivery: Segments reordered at receiver
  • Full duplex: Simultaneous bidirectional data flow
  • Flow control: Sliding window (receiver advertises rwnd)
  • Congestion control: Slow start, congestion avoidance, fast retransmit, fast recovery
// TCP Header (20 bytes minimum) | Src Port (16) | Dst Port (16) | | Sequence Number (32) | | Acknowledgment Number (32) | | HLEN | Reserved | Flags | Window (16)| | Checksum (16) | Urgent Pointer (16)| | Options (variable) | Flags: URG, ACK, PSH, RST, SYN, FIN

TCP Connection Lifecycle

3-Way Handshake (Open): Client → [SYN, seq=x] → Server Client ← [SYN+ACK, seq=y, ack=x+1] ← Server Client → [ACK, ack=y+1] → Server 4-Way Handshake (Close): Client → [FIN] → Server Client ← [ACK] ← Server Client ← [FIN] ← Server Client → [ACK] → Server (Client waits 2×MSL in TIME_WAIT state)

TCP Congestion Control

Phase Behavior
Slow Start cwnd starts at 1 MSS; doubles every RTT until ssthresh
Congestion Avoidance cwnd increases by 1 MSS per RTT (linear growth)
Fast Retransmit 3 duplicate ACKs → retransmit without waiting for timeout
Fast Recovery ssthresh = cwnd/2; cwnd = ssthresh + 3; stay in Cong. Avoidance
Timeout ssthresh = cwnd/2; cwnd = 1; restart Slow Start
