Network Types & Classifications

What is a Computer Network?

A computer network is a collection of interconnected devices (computers, printers, servers) that can share resources and communicate with each other. Networks are classified by geographic scope, transmission medium, and communication mode.

💡 Key mental model: Think of networks like postal systems. LAN = your office building's internal mail. MAN = city mail. WAN = national/international post.

🏠 LAN

  • Local Area Network
  • Covers single building / campus
  • Range: up to ~1 km
  • High speed: 100 Mbps–10 Gbps
  • Privately owned
  • Low error rates
  • e.g., Office network, college lab

🏙️ MAN

  • Metropolitan Area Network
  • Covers a city/town
  • Range: 5–50 km
  • Medium speed: 10–1000 Mbps
  • Public or private
  • e.g., Cable TV network, city Wi-Fi

🌍 WAN

  • Wide Area Network
  • Covers countries/continents
  • Range: unlimited
  • Lower speed (varies)
  • Public/leased lines
  • Higher error rates
  • e.g., The Internet, ATM network

Wired vs. Wireless Networks

WIRED

Data travels over physical cables (twisted pair, coaxial, fiber optic). More reliable, secure, and faster. Harder to install (infrastructure needed).

WIRELESS

Data travels via radio waves / microwaves / IR. More flexible and mobile. Subject to interference, eavesdropping, and range limits.

Broadcast vs. Point-to-Point

BROADCAST

One node sends data that ALL other nodes receive. They check address fields to decide relevance. Used in older Ethernet (bus), Wi-Fi.

TV transmission is a real-world analogy — one channel, everyone receives.

POINT-TO-POINT

A dedicated link connects exactly two devices. Data travels directly. Used in PPP links, fiber trunks, phone calls.

Like a phone call — direct line between you and one other person.

OSI & TCP/IP Models

The Layering Concept

Networking is complex. Layering breaks the problem into manageable pieces. Each layer serves the layer above and is served by the layer below. This means you can change how Wi-Fi works without rewriting your email application.

🧩 Analogy: Think of shipping a package. You write a letter (application) → put it in an envelope (presentation/session) → address it (network) → hand to postal service (data link) → they load it on a truck (physical). Each "layer" does its job independently.

Data Encapsulation (Super important!)

As data moves down the sending stack, each layer adds its own header (and sometimes trailer). This is called encapsulation. The receiver strips headers going back up.

Application:  [ DATA ]
Transport:    [ TCP Header | DATA ]  ← called "Segment"
Network:      [ IP Header | TCP Header | DATA ]  ← called "Packet"
Data Link:    [ MAC Header | IP Hdr | TCP Hdr | DATA | MAC Trailer ]  ← "Frame"
Physical:     101001010111000110...  ← "Bits"

History & Standards
1969

ARPANET — The First Network

US DoD-funded project. First 4-node network (UCLA, Stanford, UCSB, Utah). First message: "LO" (system crashed after 'G' in LOGIN). This is the grandfather of the Internet.

1973

Ethernet Invented — Bob Metcalfe (Xerox PARC)

The dominant LAN technology to this day. Originally 10 Mbps over coax; now 100 Gbps over fiber.

1974

TCP/IP Protocol Suite — Cerf & Kahn

The foundational paper "A Protocol for Packet Network Intercommunication." Cerf is called "Father of the Internet." TCP/IP became the universal language of networking.

1984

OSI Model Published — ISO

The Open Systems Interconnection reference model. Designed to standardize networking across vendors. Widely taught but TCP/IP won in practice.

1991

World Wide Web — Tim Berners-Lee (CERN)

HTTP and HTML brought the Internet to the masses. The Web ≠ Internet. The Internet is the infrastructure; the Web is an application on it.

1999

IEEE 802.11b Wi-Fi (11 Mbps)

Wireless networking goes mainstream. Now we have 802.11ax (Wi-Fi 6) supporting multi-gigabit speeds.

Data & Signals

Data vs. Signals

Data is the information we want to communicate (text, images, voice). Signals are the physical representation used to transmit data (electromagnetic waves).

Data can be analog (continuous — human voice) or digital (discrete — binary 0s and 1s). Signals can also be analog or digital.

📈 Analog Signal

  • Continuous waveform (sine wave)
  • Characterized by: Amplitude, Frequency, Phase
  • Can take infinite values
  • Degrades with noise (hard to recover)
  • e.g., AM/FM radio, old telephone lines

⬛ Digital Signal

  • Discrete pulses (square wave)
  • Characterized by: Bit rate, Bit interval
  • Only two values: 0 or 1
  • Easier to regenerate (repeaters)
  • e.g., Ethernet, USB, modern networks

Critical Signal Properties

TermDefinitionUnit
AmplitudeHeight/strength of the wave (signal power)Volts
FrequencyNumber of cycles per secondHz (Hertz)
PeriodTime for one complete cycle (T = 1/f)Seconds
PhasePosition of wave relative to a reference timeDegrees/Radians
WavelengthDistance covered by one cycle (λ = c/f)Meters
BandwidthRange of frequencies in a signalHz
Bit RateNumber of bits transmitted per secondbps

Theoretical Limits — Nyquist & Shannon

These two theorems are EXAM FAVORITES. Know them cold.

Nyquist Theorem — For a noiseless channel:

Max Bit Rate = 2 × B × log₂(L)  bps

Where B = bandwidth (Hz), L = number of signal levels

Shannon's Capacity Theorem — For a noisy channel:

Max Capacity = B × log₂(1 + SNR)  bps

Where SNR = Signal-to-Noise Ratio (often given in dB)

⚠️ Shannon gives the THEORETICAL MAXIMUM. Nyquist gives the limit based on signal levels. Real systems are always below Shannon capacity.

Block Coding

Used to improve data integrity. Divides data into m-bit groups, maps each to an n-bit codeword (where n > m), then applies line coding. Written as mB/nB.

Example: 4B/5B — every 4-bit block maps to a 5-bit code. The extra bit ensures no codeword has more than one leading zero and no more than two trailing zeros, enabling clock recovery.

Converting Analog to Digital (PCM)

Pulse Code Modulation (PCM) converts analog signals to digital in 3 steps:

1. Sampling

  • Sample the analog signal at regular intervals
  • Nyquist: sample at ≥ 2×max frequency
  • Voice: 4000 Hz → 8000 samples/sec

2. Quantization

  • Round each sample to nearest level
  • 256 levels = 8 bits per sample
  • Introduces quantization noise

3. Encoding

  • Represent each level as binary
  • Telephone: 8 bits × 8000 samples = 64 Kbps
  • CD audio: 44,100 Hz, 16 bits → 1.4 Mbps

Analog Transmission & Modulation

Why Do We Modulate?

Digital data must sometimes be sent over analog channels (e.g., old phone lines, radio). Modulation maps digital bits onto a carrier analog wave by varying one of its properties.

ASK — Amplitude Shift Keying

  • Change amplitude to represent bits
  • 0 = low amplitude, 1 = high amplitude
  • Simple but susceptible to noise
  • Used in optical fiber (on/off)

FSK — Frequency Shift Keying

  • Change frequency to represent bits
  • 0 = low freq, 1 = high freq
  • Less noise-sensitive than ASK
  • Used in old modems, garage door openers

PSK — Phase Shift Keying

  • Change phase to represent bits
  • BPSK: 2 phases (0°, 180°)
  • QPSK: 4 phases → 2 bits/symbol
  • Used in Wi-Fi, satellite, LTE

QAM — Quadrature Amplitude Modulation

Combines ASK + PSK. Varies both amplitude and phase. This allows more bits per symbol — very bandwidth efficient.

64-QAM: 6 bits per symbol (2⁶ = 64 combinations). 256-QAM: 8 bits per symbol. Used in cable modems, Wi-Fi 6, 4G/5G.
Bits per symbol = log₂(M)  where M = number of distinct signal states

Analog-to-Analog Modulation

For transmitting analog data over analog channels (e.g., radio broadcasting):

TypeWhat variesBandwidthUse
AM (Amplitude Modulation)Amplitude of carrier2 × signal bandwidthAM radio (535-1605 kHz)

Bandwidth Utilization & Multiplexing

What is Multiplexing?

Multiplexing allows multiple signals to share a single transmission medium simultaneously. A MUX combines signals at the sender; a DEMUX separates them at the receiver.

💡 Analogy: A highway with multiple lanes. Multiple cars (signals) travel simultaneously without collision because they use different lanes (channels). Multiplexing = managing these lanes.

Switching Techniques

Why Switching?

In a network with many devices, we can't have direct links between every pair (too expensive, mesh problem). Switching lets us route data through intermediate nodes on a shared infrastructure.

📞 Circuit Switching

  • Dedicated path established BEFORE data transfer
  • Path reserved for entire duration
  • Three phases: Setup → Transfer → Teardown
  • Guaranteed bandwidth
  • Wasteful if idle (resources locked)
  • e.g., Traditional telephone (PSTN)

📬 Message Switching

  • Entire message stored at each node
  • "Store-and-forward" switching
  • No dedicated path
  • Messages can be queued (delayed)
  • Better utilization than circuit
  • e.g., Email (conceptually), telegrams

📦 Packet Switching

  • Message split into fixed-size packets
  • Each packet routed independently
  • Packets may arrive out of order
  • Excellent link utilization
  • Internet standard
  • e.g., Internet (IP), ATM
CIRCUIT SWITCHING:
A ────────────────────────────────── B
    [Setup][====DATA FLOWS====][End]
    (path reserved, no sharing)

MESSAGE SWITCHING:
A → Node1 → Node2 → Node3 → B
    [store] [store] [store]
    (whole message hops, may delay)

PACKET SWITCHING (Datagram):
Packet 1: A → N1 → N3 → B
Packet 2: A → N1 → N2 → N4 → B  ← different path!
Packet 3: A → N3 → B
          (each packet routed independently)

Data Link Layer — Wired

What Does the Data Link Layer Do?

The DLL is responsible for node-to-node delivery — getting a frame from one hop to the next. It sits between the Physical Layer (raw bits) and the Network Layer (IP packets). It has two sub-layers:

LLC — Logical Link Control

  • Upper sub-layer
  • Error control and flow control
  • Multiplexing of higher-layer protocols
  • IEEE 802.2

MAC — Medium Access Control

  • Lower sub-layer
  • Controls who sends on shared medium
  • MAC addresses (48-bit hardware address)
  • Framing, error detection

Key DLL Functions

FunctionDescription
FramingDivide bitstream into discrete frames with header/trailer for delineation
Error DetectionDetect corrupted frames using CRC, checksum, parity
Error CorrectionRetransmit or correct errors (ARQ protocols)
Flow ControlPrevent fast sender from overwhelming slow receiver
MACManage access to shared medium — avoid collisions

Error Detection Methods

Parity Check

  • Add 1 bit so total 1s is even (even parity) or odd
  • Detects single bit errors
  • Can't detect 2-bit errors
  • Very simple, low overhead

CRC

  • Cyclic Redundancy Check
  • Polynomial division of data
  • Append remainder as FCS
  • Very powerful — detects all burst errors < degree of polynomial
  • Used in Ethernet, Wi-Fi

Checksum

  • Sum of all data segments
  • Send 1's complement of sum
  • Receiver: sum all + check = all 1s?
  • Used in IP, TCP, UDP headers
⚠️ Error detection only DETECTS errors. Error CORRECTION requires either retransmission (ARQ) or forward error correction (FEC) using Hamming codes or similar.

Multiple Access Protocols

The Problem: Shared Medium

When multiple devices share the same transmission medium (like a bus or Wi-Fi), collisions happen when two nodes transmit simultaneously. Multiple Access Protocols coordinate access to avoid or recover from collisions.

Three Categories

🎲 Random Access

  • No central control
  • Any node can transmit anytime
  • Collisions possible — handled by protocol
  • Examples: ALOHA, CSMA, CSMA/CD, CSMA/CA

🎫 Controlled Access

  • Nodes coordinate before transmitting
  • No collisions
  • Examples: Polling (master/slave), Token Passing, Reservation

✂️ Channelization

  • Divide channel into sub-channels
  • No collisions (separate channels)
  • Examples: FDMA, TDMA, CDMA

ALOHA → CSMA/CD Evolution

ProtocolHow it worksEfficiency
Pure ALOHASend anytime. Collision? Wait random time, resend.Max 18.4%
Slotted ALOHASend only at start of time slots. Less collision window.Max 36.8%
CSMA (1-persistent)Sense medium first. If idle, send immediately. Collision still possible.Better

CSMA/CD — Binary Exponential Backoff

After collision k: wait random slots in range [0, 2ᵏ - 1]. After 16 collisions, give up.

1st collision:  wait 0 or 1 slots      (range: 0 to 1)
2nd collision:  wait 0,1,2, or 3       (range: 0 to 3)
3rd collision:  wait 0 to 7            (range: 0 to 7)
kth collision:  wait 0 to 2ᵏ-1 slots
Max k = 10, then range stays at 0-1023
After 16 failures → abort, report error
Minimum frame size in Ethernet = 64 bytes. This ensures a collision can be detected before transmission ends. If frame is shorter, collision may go undetected.

Data Link Layer — Wireless

Why Wireless is Harder

Hidden Node Problem

  • Node A and C both in range of B, but not each other
  • A can't hear C, so A doesn't know C is transmitting
  • Both transmit to B → collision at B
  • Solution: RTS/CTS handshake (Request to Send / Clear to Send)

Exposed Node Problem

  • Node B transmits to A. C hears B and thinks medium is busy.
  • C wants to transmit to D (out of A's range) — this would be fine!
  • C unnecessarily waits → wasted capacity
  • More complex to solve

IEEE 802.11 CSMA/CA Protocol

Sender wants to transmit:
  1. Sense medium — busy? Wait until idle
  2. Wait for DIFS (Distributed Inter-Frame Space) 
  3. Pick random backoff counter (in contention window)
  4. Countdown during idle periods, pause when medium busy
  5. When counter = 0: send RTS (Request to Send)
  6. Receiver sends CTS (Clear to Send) after SIFS
  7. Sender transmits data frame
  8. Receiver sends ACK after SIFS
  
  DIFS: longer wait (before competing)
  SIFS: shorter wait (for ACK/CTS — high priority)

Bluetooth (IEEE 802.15.1)

Short-range wireless PAN (Personal Area Network) technology.

Key Features

  • Frequency: 2.4 GHz ISM band
  • Range: ~10m (Class 2), up to 100m (Class 1)
  • Uses FHSS: hops 1600 times/second across 79 channels
  • Piconet: 1 master + up to 7 active slaves
  • Scatternet: overlapping piconets

Versions

  • BT 1.x: 1 Mbps
  • BT 2.0 + EDR: 3 Mbps
  • BT 3.0: 24 Mbps (uses 802.11)
  • BT 4.x: BLE (Bluetooth Low Energy) — IoT
  • BT 5.x: 2 Mbps, 4× range, IoT focus

WiMAX (IEEE 802.16)

Worldwide Interoperability for Microwave Access. A wireless broadband standard designed for metropolitan area networks.

  • Range: up to 50 km (line of sight) or 5-15 km (NLOS)
  • Speed: up to 70 Mbps (802.16e up to 30 Mbps mobile)
  • Frequencies: 2–11 GHz (licensed and unlicensed)
  • Uses OFDM for multipath resistance
  • Largely displaced by LTE/4G in practice

Mobile IP

When a device moves across networks, its IP address changes, breaking existing connections. Mobile IP solves this with two IP addresses:

Home Address (HoA)

  • Permanent address
  • What others use to reach you
  • Registered at Home Agent (HA)

Care-of Address (CoA)

  • Temporary address at visited network
  • Registered with Home Agent
  • HA tunnels packets to CoA
Process: Correspondent → Home Agent → (IP tunnel) → Care-of Address → Mobile Node. Triangle routing is a drawback — packets travel longer path than necessary.

Transport Layer — ARQ Protocols

Automatic Repeat Request (ARQ)

ARQ protocols handle reliable data transfer over an unreliable channel. They use acknowledgments (ACKs) and retransmissions to ensure every packet is delivered.

Key metric: Efficiency (η) = throughput / maximum possible throughput. Depends on propagation delay and window size.

Stop-and-Wait (Simplest)

Send one frame. Wait for ACK. If ACK received → send next. If timeout → retransmit.

Sender:    [Frame 0]─────────────────▶
                                      [ACK 0]◀───
           [Frame 1]─────────────────▶
                                      [ACK 1]◀───

If timeout (no ACK): retransmit same frame.
Efficiency (η) = 1 / (1 + 2a)    where a = Propagation delay / Transmission time
⚠️ Very inefficient at high bandwidth-delay products. If propagation delay is large, sender is idle waiting for ACK most of the time.

Sliding Window Protocols

Send multiple frames before waiting for ACKs. The window size (W) is the number of outstanding unacknowledged frames allowed.

Efficiency = W / (1 + 2a)  if W < 1+2a  else η = 1 (full utilization)

Go-Back-N (GBN)

  • Window size: up to 2ⁿ⁻¹
  • Error → retransmit that frame AND all subsequent frames
  • Simpler receiver (no buffering)
  • Wastes bandwidth when errors occur
  • Receiver only accepts in-order frames

Selective Repeat (SR)

  • Window size: up to 2ⁿ⁻¹ (for sender AND receiver)
  • Error → retransmit ONLY that specific frame
  • Receiver buffers out-of-order frames
  • More efficient, more complex
  • Used when channel has high error rate
GBN (window=4, frame 2 lost):
Sender: [0][1][2][3][4]...   sends 4 at a time
Error on 2:
Receiver: accepts 0,1; discards 3,4 (out of order)
Sender retransmits: [2][3][4][5]...  ← retransmit from 2

SR (window=4, frame 2 lost):
Receiver: accepts 0,1,buffers 3,4
Sender retransmits: only [2]
Receiver: delivers 2,3,4 in order

Bidirectional — Piggybacking

In full-duplex links, ACKs can be "piggybacked" on data frames going in the other direction. Instead of sending a separate ACK frame, the receiver attaches the ACK to the next data frame it sends. Reduces overhead significantly.

UDP & TCP

Transport Layer Role

The Transport Layer provides process-to-process delivery (not just host-to-host like IP). It uses port numbers to identify which application to deliver data to. Two main protocols: UDP and TCP.

UDP — User Datagram Protocol

  • Connectionless — no setup
  • Unreliable — no ACKs, no retransmit
  • No flow control, no congestion control
  • Very fast, very low overhead
  • 8-byte header only (vs TCP's 20+ bytes)
  • Good for: DNS, video streaming, VoIP, gaming, DHCP
  • "Fire and forget"

TCP — Transmission Control Protocol

  • Connection-oriented (3-way handshake)
  • Reliable — ACKs, retransmission
  • Flow control (receiver window)
  • Congestion control (AIMD)
  • In-order delivery guaranteed
  • 20-byte header minimum
  • Good for: HTTP, FTP, SSH, email

Flow Control & Congestion Control

Flow Control (Receiver-side)

  • Receiver advertises window size (rwnd)
  • Sender can't have more than rwnd bytes unacknowledged
  • Prevents overwhelming slow receiver
  • Window = 0? Sender pauses

Congestion Control (Network-side)

  • Slow Start: cwnd starts at 1 MSS, doubles each RTT
  • Congestion Avoidance: linear increase once threshold hit
  • On loss: threshold halved, cwnd reset
  • AIMD: Additive Increase, Multiplicative Decrease
Effective send rate = min(rwnd, cwnd) — TCP respects BOTH flow control AND congestion control simultaneously.
